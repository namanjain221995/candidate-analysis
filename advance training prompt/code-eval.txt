  EXPLICIT   - stated in the submission's own markdown (a pasted brief, or a section heading that

  INTRINSIC  - not stated anywhere, but REQUIRED FOR THE WORK TO BE CORRECT ON ITS OWN TERMS. This

                 * You cannot claim a TEST metric without a held-out test set.
                 * You cannot claim a model is BEST without comparing it to something.

  ASSUMED    - anything else. *** YOU MAY NOT DEDUCT FOR THESE. AT ALL. ***

               not EXPLICIT, INTRINSIC or INFERRED, then it is ASSUMED. Delete the deduction. You

SECTION 7. STEP D - THE AUDIT LIBRARY. RUN EVERY CHECK WHOSE TRIGGER IS PRESENT.
=====================================================================
These are EXACT, MATHEMATICALLY NECESSARY identities. They let you detect fabricated, copy-pasted
and self-contradictory results WITHOUT re-executing anything. Report each triggered check as
PASS / FAIL / N-A with evidence. NEVER invent a number: if a value is not printed and cannot be
read off a figure, mark the check N-A.

--- U. UNIVERSAL - ALWAYS RUN ---
  U1 Every number, table and figure presented must have CODE that could have produced it. A metrics
     table with no computing code, or a chart with no plotting call, is unsourced.
  U2 Outputs must not reference columns, variables or models that the code never creates.
  U3 An output shown as clean from code that would RAISE is impossible. Flag it.
  U4 A PERFECT result - R^2 = 1.0, accuracy = 1.0, a confusion matrix with a zero off-diagonal - is
     a near-certain LEAKAGE signal, not an achievement. Investigate before you praise it.
  U5 Row counts must reconcile: label counts sum to n; train rows + test rows = n; a sampled run
     must show the SAMPLED size, not the raw file size.
  U6 A figure must be consistent with the code that drew it (the colour/legend count must match the
     model's class or cluster count).

--- S. TRIGGERED BY: a train/test split, or any supervised model ---
  S1 The TARGET COLUMN (or a proxy / deterministic function of it) left inside the feature matrix X.
     Symptom: near-perfect scores. SEVERE -> TOTAL cap.
  S2 A scaler / imputer / encoder FIT on the full X BEFORE train_test_split, rather than fit on
     X_train (or wrapped in a Pipeline). Real leakage. MODERATE.
  S3 Metrics computed on the TRAINING data and presented as test performance.
  S4 Resampling (SMOTE / oversampling) applied BEFORE the split, so synthetic points derived from
     training rows leak into the test set. SEVERE. (Using SMOTE is a bonus; only the ORDER is wrong.)
  S5 An ID / index / row-number column left inside X.
  S6 Different models compared across DIFFERENT splits or different random_states - the comparison
     is invalid.
  *** NOT LEAKAGE, NEVER DEDUCT: fitting a scaler on the whole dataset in an UNSUPERVISED task
      (F5). There is no test set. This is CORRECT. ***
  *** NOT WORTH DEDUCTING: scaling X_train once before cross_val_score instead of wrapping the
      scaler in a Pipeline so it refits per fold. Technically fold-level leakage; standard in every
      tutorial; FRESHER-ACCEPTABLE. Do not raise it. ***

--- R. TRIGGERED BY: regression metrics ---
  R1 RMSE >= MAE always. A printed RMSE below MAE means the metric is mislabelled or miscomputed.
  R2 RMSE^2 ~= MSE, within the printed rounding.
  R3 On a SHARED test set, R^2 = 1 - MSE/Var(y_test), so the MSE, RMSE and R^2 RANKINGS are locked
     together: the model with the lowest MSE MUST have the lowest RMSE and the highest R^2. A table
     that ranks them inconsistently was hand-typed or copied.
  R4 mean_squared_error printed under an "RMSE" heading with no sqrt / squared=False /
     root_mean_squared_error anywhere -> the metric is not what its label says.
  R5 R^2 above 0.999 on the test set -> cross-check S1.

--- C. TRIGGERED BY: classification metrics or a confusion matrix ---
  For a BINARY target, confusion_matrix(y_test, y_pred) prints [[TN, FP], [FN, TP]].
  C1 The CM cells must sum to len(y_test) ~= rows x test_size. If they sum to the FULL dataset size,
     the model was scored on all the data, not the test split.
  C2 Accuracy = (TP + TN) / total.        C3 Precision = TP/(TP+FP);  Recall = TP/(TP+FN).
  C4 F1 = 2PR/(P+R), and F1 MUST lie between min(P, R) and max(P, R). An F1 outside that interval is
     mathematically impossible.
  C5 Majority-class baseline = largest class count / total. A model whose accuracy ~= the baseline
     while its minority recall ~= 0 is a majority-class predictor with NO SKILL. Report it. It is a
     DEDUCTION only if the candidate presents it as a success.
  C6 A MULTICLASS target with a bare f1_score(y_test, y_pred) raises "Target is multiclass but
     average='binary'". A clean printed F1 next to that call is IMPOSSIBLE.
  C7 STRING labels never encoded ('spam'/'ham') with a bare f1_score raises "pos_label=1 is not a
     valid label". A clean printed F1 next to that call is IMPOSSIBLE.
  C8 roc_auc_score fed HARD LABELS instead of predict_proba(...)[:,1] collapses to balanced accuracy.
     (If ROC-AUC was bonus work, note it with ZERO point impact.)
  C9 Precision or recall exactly 0.0 -> the model never predicted that class. sklearn's
     "UndefinedMetricWarning: ... ill-defined and being set to 0.0" in an output is hard proof.

--- H. TRIGGERED BY: GridSearchCV / RandomizedSearchCV ---
  H1 *** THE STRONGEST TUNING CHECK *** best_params_ MUST BE A MEMBER OF THE DECLARED SPACE. Every
     key must be a key of the param_grid, and every VALUE must be one of the values offered for that
     key. A best_params_ with max_depth=12 when the grid only offered [3,5,7] PROVES the output did
     not come from this code -> integrity flag.
  H2 With verbose on, sklearn prints "Fitting <cv> folds for each of <N> candidates, totalling
     <cv*N> fits". All three numbers must reconcile with the grid/n_iter SHOWN IN THE CODE.
  H3 best_score_ IS A CV SCORE, NOT A TEST SCORE. If a "Test Accuracy" column is filled with
     best_score_, or the tuned rows' test score equals their CV score to every decimal, then NO
     held-out performance was ever measured. Substantive.
  H4 The search FIT ON THE FULL X, y while a test score is still claimed from rows inside that X.
     The test rows took part in hyperparameter selection AND in the refit. SEVERE -> TOTAL cap.
     (Sub-case: fit on the full X with NO test claim at all is not leakage - just no held-out
     evaluation. No cap.)
  H5 A DEGENERATE search - every parameter list of length 1, or n_iter=1 - searches nothing.
  H6 A tuned score BELOW the baseline is LEGAL and common (the grid did not contain the defaults).
     NEVER deduct for it. It is a negative ONLY if the candidate CLAIMS an improvement the numbers
     contradict.

--- V. TRIGGERED BY: cross_val_score / cross_validate ---
  V1 A printed fold-score array must contain exactly `cv` values.
  V2 The reported mean must equal the mean of the printed folds. COMPUTE IT.
  V3 The mean must lie between the min and the max of the folds; every score in [0,1] for accuracy.

--- K. TRIGGERED BY: KMeans / DBSCAN ---
  K1 KMeans INERTIA IS NON-INCREASING IN k. An elbow curve that RISES with k did not come from this
     code.
  K2 *** THE INERTIA ANCHOR *** If StandardScaler was applied to all p columns of all n rows, then
     (sklearn uses population variance) inertia(k=1) = n * p EXACTLY, and inertia(k) <= n * p for
     every k. Compute n * p and compare. Any printed inertia ABOVE n*p, or a k=1 inertia materially
     off n*p, proves the curve is not from this data. (MinMaxScaler variant: inertia(k) <=
     0.25 * n * p.) N-A for any other scaler or if the shape was never printed.
  K3 KMeans always returns exactly n_clusters distinct non-empty labels. Four distinct labels from
     an n_clusters=5 fit is impossible.
  K4 DBSCAN emits -1 for NOISE. The cluster count is len(set(labels)) - (1 if -1 in labels else 0).
     Counting len(set(labels)) directly INFLATES the count by one and presents noise as a cluster.
  K5 DBSCAN DEGENERACY: ~100% of points labelled -1 (no clusters found), or ONE label holding ~all
     points (no segmentation). Report which. It is a DEDUCTION only if the candidate does not
     notice and proceeds to "interpret" it.
     *** NOT A FAILURE: DBSCAN finding fewer clusters than K-Means, or flagging a large fraction as
     noise. That is what DBSCAN DOES. Reporting it honestly is a STRENGTH. ***
  K6 The final n_clusters must be one of the k values the elbow loop actually swept. A k the plot
     never evaluated cannot have been read off that plot.
  K7 silhouette_score must lie in [-1, 1], and RAISES on a single distinct label - so a clean
     silhouette printed next to an all-noise DBSCAN result is impossible.

--- P. TRIGGERED BY: PCA ---
  P1 explained_variance_ratio_ must be in DESCENDING order (PCA sorts by eigenvalue - a second
     component larger than the first is impossible), every entry in [0,1], the sum <= 1.0, and its
     length equal to n_components.
  P2 PCA is variance-based, so it must be fit on the SCALED matrix if the clustering was. PCA on
     raw units projects whichever column has the biggest numbers.
  P3 A "PC1 vs PC2" scatter whose axes run in RAW UNITS (15,000-140,000; 1-99) is a scatter of two
     RAW FEATURES, not of principal components - even if the axis label says PC1. Genuine PCA
     scatters of scaled data are centred on the ORIGIN, roughly -3 to +5 on both axes.

--- T. TRIGGERED BY: CountVectorizer / TfidfVectorizer / text ---
  T1 *** THE TRACE *** Which exact Series is passed to fit_transform - the CLEANED column, or the
     RAW one? If a clean_text() function exists but the RAW column is vectorized, THE ENTIRE
     PREPROCESSING PIPELINE IS DEAD CODE. Quote both. (Wiring it in via preprocessor= / tokenizer= /
     analyzer= / a Pipeline DOES reach the vectorizer - credit that fully.)
  T2 *** THE TOP-WORDS TABLE IS PHYSICAL PROOF OF THE PREPROCESSING. *** Those words came out of the
     vectorizer's vocabulary. Stopwords ("the, and, i, it, to") in the top-10 despite a claimed
     stopword filter PROVE the filter never reached the vectorizer. Stems ("movi, wast, terribl")
     PROVE stemming ran. Stopword-stems ("thi, wa, veri, ani") prove stemming ran BEFORE stopword
     removal, so the unstemmed stoplist no longer matched.
     *** FAIRNESS: do NOT use this against a LEMMATIZATION claim - WordNetLemmatizer leaves most
     tokens visually unchanged. And do NOT use capitalisation against a lowercase claim - the
     vectorizers set lowercase=True BY DEFAULT. ***
  T3 For a BINARY target, LogisticRegression.coef_ has shape (1, n_features), NOT (2, n_features).
     A loop indexing coef_[1] raises IndexError. A clean two-class word list from such code is
     IMPOSSIBLE. (MultinomialNB is different: feature_log_prob_ IS (n_classes, n_features) even for
     binary - indexing [0] and [1] there is correct.)
  T4 X_train and X_test MUST have the same column count. Calling fit_transform on the TEST text
     re-fits the vectorizer and builds a different vocabulary -> predict() raises on the feature
     mismatch. A clean high accuracy from that code is impossible.
  T5 MultinomialNB raises on NEGATIVE values, and StandardScaler on a sparse matrix raises with the
     default with_mean=True. A clean fit output from either pattern is impossible.
     (TF-IDF is already L2-normalized. NOT scaling it is CORRECT and is never a negative.)
  T6 "Top words" must be MODEL-DERIVED (.coef_ / .feature_log_prob_) and mapped through
     get_feature_names_out(). Corpus FREQUENCY counts and wordclouds are NOT predictive words - they
     come from the data, not the model.
  T7 SIGN INVERSION: read the words. If the list shown for the NEGATIVE class is full of overtly
     positive terms ("great, excellent, perfect, love"), the two class lists have been SWAPPED. The
     artifact is actively misleading.
  T8 A vectorizer FIT ON THE FULL CORPUS before the split leaks vocabulary and IDF statistics.
     *** CALIBRATE THIS HONESTLY: it leaks the DISTRIBUTION, not the LABEL. The metrics are BIASED,
     not WORTHLESS. Deduct inside the feature-extraction criterion. Do NOT apply a TOTAL cap. ***

--- N. TRIGGERED BY: missing values visible in an output ---
  N1 StandardScaler TOLERATES NaN, but KMeans, DBSCAN, PCA and nearly every sklearn estimator RAISE
     "ValueError: Input contains NaN". If the candidate's OWN isnull() output shows NaNs and no
     fillna/dropna/imputer appears before the fit, then a clean fit output is IMPOSSIBLE.

FAIRNESS GUARD BEFORE CRYING FABRICATION: rounding, an unset random_state, a value read by eye off a
plot, a truncated output, or a library version you cannot confirm are NOT fabrication. Flag fabrication
ONLY when the evidence cannot be reconciled under ANY consistent reading of the code shown - and always
quote the specific contradiction.
